M1 Mac CobraPy Install Guide

By Filip Casey (April 2022)

When attempting to install the CobraPy python package on an M1 Mac, it is likely that you’ll run into a legacy-install-failure error. This seems to be due to 2 outdated packages in the CobraPy package (swiglpk and libsbml), until these issues are dealt with, this workaround should allow for the package to work on M1 Macs.

1. Install Conda

If you are not currently running an Anaconda distribution, you’ll need to install it. For the purposes of this guide MiniConda3 works fine.

2. Create a new Conda environment

This step will allow for a python interpreter to run on your system using the Rosetta x64 emulator, instead of the ARM architecture of M1 Macs. Taken from this Github Post.

Run this code in your terminal:

CONDA_SUBDIR=osx-64 conda create -n rosetta python   # create a new environment called rosetta with intel packages.
conda activate rosetta
python -c "import platform;print(platform.machine())"
conda env config vars set CONDA_SUBDIR=osx-64  # make sure that conda commands in this environment use intel packages
conda deactivate rosetta
conda activate rosetta
echo "CONDA_SUBDIR: $CONDA_SUBDIR"

This will create a new conda environment called rosetta, that will run using Rosetta.

3. Install Cobra

Inside the rosetta environment, call conda install -c conda-forge cobra . You can tell that you are in the rosetta environment if your terminal looks something like this: 

(rosetta) USERNAME@COMPUTER ~ %
